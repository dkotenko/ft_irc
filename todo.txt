2. Реализовать механизм валидации:
	под "прекратить обработку" подразумевается возврат пустого MessageOutput, либо MessageOutput с сообщением ошибки, либо с флагом mustBeSent

	- серверу приходит строка
	- парсер раскладывает сроку в MessageInput
	- Проверка comand: есть словарь команд. Если в словаре нет команды - прекратить обработку
	- все функции обработки имеют одинаковый интерфейс: void handle_*название_команды*(MessageInput messageInput);
	- При создании сервера (парсера) инициализируется словарь функций обработки команд
	- Обращение к нужной функции: handleMap[messageInput.comand](messageInput);
	- в каждой handle функции происходит валидация соответствующей команды. Ошибка валидации - прекратить обработку
	- если парсеру следует сообщить об ошибке клиенту, сообщение ошибки заполняется внутри функции handle_error_message(MessageInput messageInput)
	- если парсеру следует сообщить об ошибке серверу, сообщение заполняется кодом ошибки в соответствующем поле
	- при успешной валидации соответствующие действия в логике вызываются из handle_*название_команды*

3. Согласно документации, максимальный размер сообщения клиента 512 байт включая название команды, параметров и символов конца строки \r\n
4. Channels names are strings (beginning with a '&' or '#' character) of length up to 200 characters.

	For the embedded work, GDB needs to be set for your selected Preferences/Toolchains.
	For debugging console apps LLDB needs to be selected in Preferences/Toolchains.
